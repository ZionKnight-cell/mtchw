# Mtchw PRD

## App Name

Mtchw

## One-Sentence Summary

Mtchw is a simple mobile app that gives users one small thing to do when they are bored, tired of scrolling, tired of games, tired of YouTube, but still want to use their phone.

## Main Problem

People often open their phones when they are bored but end up scrolling social media, watching videos, or wasting time without feeling satisfied.

Mtchw helps by giving the user one tiny, low-pressure activity to do instead.

## Target User

Mtchw is for people who:

- Feel bored
- Feel restless
- Are tired of scrolling
- Want something small to do
- Do not want a serious productivity app
- Do not want a social media app
- Do not want a complicated habit tracker

## Emotional Moment

The app is designed for the moment when someone thinks:

“I’m bored. I want to do something, but I don’t know what.”

## MVP Goal

The first version should let the user:

1. Open the app
2. Tap “I’m bored”
3. Choose energy level
4. Choose time available
5. Choose activity category
6. Receive one small activity
7. Tap Done, Skip, Save, or More Like This
8. View saved activities
9. View activity history
10. Use the app mostly offline

## MVP Features

### 1. Home Screen

The home screen should have:

- App name: Mtchw
- Short tagline
- Large “I’m bored” button
- Small links to Saved, History, and Settings

### 2. Energy Selection

The user should be able to choose:

- Low energy
- Medium energy
- High energy
- Surprise me

### 3. Time Selection

The user should be able to choose:

- 1 minute
- 5 minutes
- 10 minutes
- 15+ minutes
- Any

### 4. Activity Category Selection

The user should be able to choose:

- Fun
- Useful
- Creative
- Calm
- Learn
- Social
- Random

### 5. Activity Card

The app should show one activity at a time.

Each activity card should show:

- Activity title
- Short instruction
- Category
- Energy level
- Estimated time

### 6. Activity Actions

Each activity should have these buttons:

- Done
- Skip
- Save
- More Like This

### 7. Saved Activities

The user should be able to save activities and view them later.

### 8. Activity History

The app should keep a simple local history of activities shown, completed, skipped, and saved.

### 9. Settings

Settings should include:

- Clear saved activities
- Clear history
- Reset app data
- Basic privacy note
- About the app

## Offline-First Requirement

The MVP should work mostly offline.

The activity database should be built into the app.

No account, login, backend, or internet connection should be required for the first version.

## Data to Store Locally

The app should store:

- Saved activities
- Activity history
- Completed activities
- Skipped activities
- Last selected preferences
- Basic settings

## Out of Scope for MVP

Do not build these in version 1:

- Login
- Signup
- Social feed
- Chat
- AI-generated suggestions
- Payments
- Subscriptions
- Leaderboards
- Complex streaks
- Cloud sync
- Notifications
- Backend dashboard

## Design Direction

The app should feel:

- Clean
- Friendly
- Slightly playful
- Low-pressure
- Calm
- Simple
- Not childish
- Not too serious

## Suggested First Build

The first tiny working version should include:

- Home screen
- “I’m bored” button
- Built-in activity database
- One activity card
- Skip button
- Save button
- Saved activities screen
- Local storage

After that works, add:

- Energy selection
- Time selection
- Category selection
- History
- Settings
- Android packaging
